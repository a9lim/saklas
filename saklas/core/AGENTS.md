# core/

Engine layer: model loading, vector extraction, steering hooks, trait monitoring, session orchestration, the generation loop, and the loom conversation tree.

## model.py

HF causal LM loading. `_LAYER_ACCESSORS` maps `model_type` ‚Üí layer-list accessor (`def`s, not lambdas). `_TESTED_ARCHS` frozenset gates a one-time `UserWarning` (tracked in `_warned`) when `model_type` isn't known-working. Cascading fallbacks on attention impl (SDPA ‚Üí eager), dtype, and device. `_compile_with_probe` wraps `torch.compile` with a 2-token prefill+decode warmup forward so inductor / Triton failures surface at load time as a caught `UserWarning` + eager fallback rather than a runtime segfault ‚ÄĒ `use_static_cuda_launcher` is forced off so kernel-arg-mismatch bugs (observed on Gemma-4 + torch 2.12) raise `TypeError` instead of crashing the process. `session.from_pretrained` uses the same helper after its `cuda_graphs` probe resolves the effective compile mode. `_load_text_from_multimodal` extracts text-only sub-models (Ministral-as-Mistral3), strips `language_model.` prefixes, dequantizes FP8 inline. `patch_torch_for_mps()` installs two process-global MPS workarounds lazily, MPS-only: `torch.histc` casts integer inputs to float for MoE routing, and `torch.ldexp` round-trips MXFP4 dequant through CPU while honoring `out=`.

## vectors.py

The low-level extraction primitives. One forward pass per prompt; `_capture_all_hidden_states` hooks every layer in a single pass. `_encode_and_capture_all` pools from the **last content token**, walking back past both `tokenizer.all_special_ids` *and* `tokenizer.added_tokens_encoder` to skip trailing chat-template markers. The `added_tokens_encoder` arm matters for tokenizers that don't promote chat-boundary tokens to `all_special_ids` (talkie's `<|user|>`/`<|end|>`/`<|assistant|>`) ‚ÄĒ without it, extraction pools at the structural turn marker where outlier channels dominate and bakes wildly oversized probe magnitudes. Contrastive diffs are cast to **fp32** before differencing (fp16 loses precision on close vectors).

`special_token_ids(tokenizer)` (= `all_special_ids` ‚ą™ `added_tokens_encoder` values) and `last_content_index(token_ids, tokenizer)` (walk back from the final position past that set, floor at 0) are the **one canonical definition** of "last non-special token" ‚ÄĒ every site that reports a single pooled hidden state shares them: `_encode_and_capture_all` (extraction), `TraitMonitor.score_per_token` + `session._score_incremental` (vector aggregate), and `ManifoldMonitor.score_aggregate` (manifold aggregate). Keep new aggregate readouts pooling through `last_content_index` rather than re-deriving the walkback (or, worse, meaning across the trajectory) so the discipline can't drift per-site.

Two extractors share scaffolding. `_capture_diffs_for_pairs` runs the contrastive forward-pass loop and returns diffs plus per-layer pos/neg running means and SAE layer set; `_share_bake_and_warn` applies the DLS keep set, share-bakes retained layers, and emits the diagnostics warning. Both extractors close over these helpers; only the per-layer direction differs.

- `extract_difference_of_means` ‚ÄĒ per-layer direction = `mean(diffs)` in fp32. Score = `||direction||_M / ref_norm` when a `whitener` is passed (Mahalanobis bake), `||direction||_2 / ref_norm` otherwise. No SVD.
- `extract_contrastive` ‚ÄĒ first principal component per layer (one batched `torch.linalg.svd` across all layers), oriented by majority vote on `diff @ direction`. Score = explained-variance ratio (or `diff_norm / activation_norm` for single-pair). Whitener is ignored (EVR is metric-invariant).

Both accept `sae=<SaeBackend>` to run extraction in SAE feature space (encode pos/neg stacks, compute the direction, decode back). Both return `(profile, diagnostics)` ‚ÄĒ `dict[int, Tensor]` over retained layers plus per-layer probe-quality metrics on the same keys.

Share-baking: each direction is scaled to its layer's mean activation norm, then multiplied by `score_i / sum(scores)` over the DLS-retained subset. The angular hook reads `||baked_L||` back out as the layer share at apply time; the additive hook absorbs share into magnitude.

`compute_dls_mask(mu_pos, mu_neg, directions, layer_means)` ‚ÄĒ discriminative layer selection (Selective Steering, Dang & Ngo 2026 Eq. 9). A layer is retained iff `(őľ_pos ‚ąí őľ_neutral)¬∑dŐā` and `(őľ_neg ‚ąí őľ_neutral)¬∑dŐā` have opposite signs; same-side layers encode concept *intensity*, not *polarity*. `dls=False` or `layer_means=None` skips the mask; if every layer fails, the helper warns and keeps all rather than emptying the profile.

`_compute_layer_diagnostics` returns five floats per layer: `evr`, `intra_pair_variance_mean`/`_std`, `inter_pair_alignment` (mean off-diagonal `|cos|` of unit diffs), `diff_principal_projection`. They persist through `save_profile` ‚Üí sidecar `diagnostics_by_layer` ‚Üí `Profile.metadata["diagnostics"]` ‚Üí `Profile.diagnostics`. `_emit_diagnostics_warning` fires once per extract when medians cross `_DIAG_DEGENERATE_EVR=0.95` + `_DIAG_DEGENERATE_INTRA_VAR=0.01` (one-sided / repetitive pair sets); never raises. The previous low-inter-pair-alignment branch was removed ‚ÄĒ at real hidden dims (4096+) natural contrastive pairs always land below any meaningful alignment threshold by sheer geometry (`E[|cos|] ‚Čą 1/‚ąöd`), so the warning fired on virtually every concept without being actionable. All four diagnostic values still persist and are surfaced via `saklas vector why`.

`compute_layer_means` averages neutral prompts for probe centering; `compute_neutral_activations` caches the per-prompt stack the whitener needs. `save_profile` / `load_profile` handle the safetensors+sidecar round-trip; `load_profile` dispatches on extension. `_template_overhead_cache` is keyed by `id(tokenizer)` ‚ÄĒ safe only while one tokenizer lives the session lifetime.

`project_profile(base, onto, operator, *, whitener=None)` ‚ÄĒ per-layer projection used by `session._materialize_projections`. `operator="~"` keeps the component aligned with `onto`, `"|"` keeps the orthogonal component. `whitener=None` ‚Üí plain Gram-Schmidt; with a whitener ‚Üí the closed-form LEACE projector for a single direction (Belrose et al. 2023). Layers absent from the whitener fall back to Euclidean per layer.

## extraction.py

`ExtractionPipeline` ‚ÄĒ the custom-concept orchestration lifted out of `session.py`. Owns the full flow: tensor cache ‚Üí curated/local `statements.json` (reused by default) ‚Üí generate scenarios ‚Üí save ‚Üí generate pairs ‚Üí save ‚Üí extract ‚Üí save tensor. Dependencies are passed structurally via two runtime-checkable protocols, `ModelHandle` and `PackWriter`; `SaklasSession` satisfies both, so construction reads `ExtractionPipeline(self, self, self.events)`. `DEFAULT_EXTRACTION_METHOD = "dim"`. `_extractor_for` dispatches through `globals()` so module-scope test monkeypatches reach it; `_method_label` produces the sidecar `method` string. Cache-hit short-circuits defer `layer_means`/`whitener` resolution (both can trigger a lazy neutral-activation build) until after the tensor-cache check via `_resolve_extract_kwargs`. `extract()` emits `VectorExtracted`.

`ManifoldExtractionPipeline` is the manifold-fitting analogue ‚ÄĒ a separate class (manifold input is N labeled node groups, not contrastive pairs). `fit(folder, *, sae=...)` loads a `ManifoldFolder`, short-circuits on a tensor-cache hit (`nodes_sha256` match ‚ÄĒ the hash folds in the corpus plus either `{domain, node_coords}` for authored folders or `{fit_mode, hyperparams}` for discover folders, so any fit-affecting edit invalidates it), else pools per-node centroids, embeds the authoring coords through the domain, fits per-layer subspaces, `save_manifold`s, refreshes the folder manifest, emits `ManifoldExtracted`. The `--sae` path reconstructs centroids through the SAE before the fit. Takes only `ModelHandle` (no `PackWriter` ‚ÄĒ manifolds own their folder type).

Discover-mode branch: when `folder.fit_mode in {"pca", "spectral"}`, the pipeline picks a reference layer (middle of the fit-layer set by default, or a `reference_layer` hyperparam override), stacks the per-node centroids at that layer (`(K, D)` fp32 CPU), runs them through `discover_coords(centroids, method=fit_mode, **hyperparams)` from `core/manifold.py`, wraps the derived coords in `CustomDomain(k)` with identity embedding, and proceeds through `fit_layer_subspace` unchanged. The sidecar metadata gains `fit_mode` + a resolved `hyperparams` block (data-driven defaults like spectral's `k_nn`/`bandwidth` are filled back in) + a `diagnostics` block. SAE variant reconstructs the reference-layer centroids through the SAE before the discover-coords call. `min_nodes(k)` is enforced *after* `k` is picked ‚ÄĒ if the picker lands on a `k` for which the heap is undersized, the fit raises naming both `n` and the floor.

## sae.py

SAE backend abstraction. `SaeBackend` is a runtime-checkable `Protocol`: `encode_layer`/`decode_layer`, `release`, `revision`, `layers: frozenset[int]`. `MockSaeBackend` is an identity-by-default dataclass for CPU tests with optional per-layer fn overrides. `SaeLensBackend` is the concrete adapter; `load_sae_backend(release, *, revision, model_id, device, dtype)` queries SAELens's release registry, validates base-model compatibility, resolves per-layer sae_ids (`_canonical_layer_map` picks the narrowest-width SAE per layer and warns on multiple candidates), and records `sae_ids_by_layer` for the sidecar. `sae_lens` imports are gated inside `load_sae_backend` so the module imports without the `[sae]` extra. Missing dep ‚Üí `SaeBackendImportError`; unknown release ‚Üí `SaeReleaseNotFoundError` with `difflib` near-match suggestions. Model-name matching (`_model_names_match`) is lenient ‚ÄĒ SAELens short names vs full HF ids.

## mahalanobis.py

`LayerWhitener` holds per-layer centered neutral activations `X_L ‚ąą ‚ĄĚ^(N, D)` and the small Woodbury inverse `K_L = (NőĽI + X XŠĶÄ)‚ĀĽ¬Ļ`; `apply_inv(layer, v)` computes `ő£_reg‚ĀĽ¬Ļ v = (1/őĽ)(v ‚ąí XŠĶÄ K X v)` in O(ND) without materializing D√óD. Ridge `őĽ_L = (||X_L||_F¬≤ / (N¬∑D)) ¬∑ ridge_scale` (`DEFAULT_RIDGE_SCALE = 1.0`). Built lazily from `~/.saklas/models/<id>/{layer_means,neutral_activations}.safetensors` via `from_neutral_activations` (in-memory) or `from_cache(model_id)` (disk-only). Primitives: `mahalanobis_cosine`, `leace_project`, `apply_inv`. `SaklasSession.whitener` is a lazy property; `bootstrap_probes` builds it eagerly so default extraction is end-to-end Mahalanobis. `Profile.cosine_similarity(..., whitener=)` and `project_profile(..., whitener=)` fall back to Euclidean when `whitener=None`.

## manifold.py

Arbitrary-dimensional / arbitrary-topology manifold steering math (Goodfire, arXiv 2605.05115) ‚ÄĒ pure-tensor, fp32, no session/IO coupling. `decompose(h, mean, basis) -> (h_par_c, h_perp)` is the shared centered-decomposition helper backing both the injection primitives (`subspace_replace` / `subspace_rotate`) and the read-side `ManifoldMonitor`; pure tensor, dtype-preserving, callers cast to fp32 before invoking when they want the fp32 numerics discipline. Domains: `ManifoldDomain` ABC + `BoxDomain` (open / periodic axes ‚ÄĒ box/disk, cylinder, n-torus), `SphereDomain` (S^n, chordal metric), `CustomDomain` (explicit immersion ‚ÄĒ the non-orientable escape hatch); `domain_from_spec` / `to_spec` round-trip the `manifold.json` tagged union. RBF primitives: `fit_rbf_interpolant` (the `r¬≥` polyharmonic kernel + affine polynomial, solved as a dense symmetric-*indefinite* saddle system via `torch.linalg.solve` ‚ÄĒ never Cholesky; raises a named poisedness error on affine-rank deficiency), `eval_rbf` (hot-path-safe, no host sync), `eval_rbf_jacobian` (analytic, no autograd, batch-generic `(.., m) ‚Üí (.., R, m)`). `fit_layer_subspace(centroids, node_params, *, n_components=DEFAULT_N_COMPONENTS)` centers `(K,D)` centroids, PCA-reduces to `R = min(n_components, K-1, rank)` (default `n_components=64`), normalizes the embedded node coords to the unit box (r¬≥ conditioning), fits the RBF; the `n_components` cap is exposed as the discover-mode `max_subspace_dim` hyperparam (see `_HYPERPARAMS_BY_MODE` in `io/manifolds.py`) and threaded through `ManifoldExtractionPipeline.fit` from `manifold.json::hyperparams.max_subspace_dim`. Smaller `R` gives finer-grained `subspace_replace` control at large K because each retained axis is an axis the injection can displace ‚ÄĒ bundled `personas` ships at `R=8` (matching its `picked_k=8`) to keep the per-őĪ steering effect proportional to the manifold's intrinsic structure; the default `R=64` is preserved for representational analysis where you want the full per-layer span. `LayerSubspace` holds the result, `Manifold` is the per-layer collection + the domain + node coords (the analogue of `Profile`). `Manifold.manifold_point(layer, position)` evaluates; `Manifold.tangent(layer, position)` returns the per-axis steering directions. At n=1 over an open axis the RBF reproduces the natural cubic spline exactly. `invert_parameterization(subspace, domain, query, node_coords, *, max_iter, n_restarts, damping)` is the nearest-point `s‚ĀĽ¬Ļ` ‚ÄĒ damped Gauss-Newton (Levenberg-Marquardt) minimizing `||s(p) ‚ąí q||` over authoring coords, warm-started from each query's nearest fit node(s) (the `node_coords` seeds) and projected back onto the domain via `clamp_position` after each step; batched over queries √ó restarts, so cost is independent of intrinsic dimension and the landing is continuous, not grid-quantized (replaces the former `resolution**n` grid scan ‚ÄĒ the domains no longer carry `sample_positions`). Eval-only, never the steering hot path. Two injection primitives, dispatched by `SteeringHook.injection_mode`: `subspace_replace(h, mean, basis, target, alpha)` (additive mode) blends `h`'s in-subspace component toward `target`, keeps the residual, and norm-restores `||h||` ‚ÄĒ `alpha=0` identity, `alpha=1` snaps onto the target. `subspace_rotate(h, mean, basis, target, alpha, theta_max)` (angular mode) Givens-rotates `h_par` toward `target` *within* the affine subspace plane by `őł = őĪ ¬∑ őł_max`, keeps `h_perp` untouched, and preserves `||h - mean||` exactly with no rescale; `alpha=0` identity, `alpha=1` lands `h_par` at a full `őł_max` rotation in the plane (so its centered direction matches the in-plane perpendicular `w_unit` from `h_par` toward `target` ‚ÄĒ not at `target` itself). The angular variant pairs with angular vector steering's "rotate toward dŐā" idiom and drops the norm-restore that subspace_replace papers over its linear blend with. `fit_layer_subspace` now returns `(LayerSubspace, ev_ratio)` ‚ÄĒ the per-layer explained-variance ratio `ő£ ŌÉ¬≤[:R] / ő£ ŌÉ¬≤(all)` capturing the fraction of inter-node centroid variance retained by the chosen R-dim subspace, used by additive-mode őĪ normalization (v4+ sidecar field `explained_variance_per_layer`); legacy callers can drop the second element. `compute_node_centroid` pools a node's mean activation (reuses `vectors._encode_and_capture_all`). `save_manifold` / `load_manifold` round-trip the per-model tensor (`layer_<L>.{mean,basis,node_params,rbf_weights,poly_coeffs,coord_offset,coord_scale}` + a shared bare `node_coords`) + JSON sidecar (now carrying `explained_variance_per_layer` when populated).

Naturalness-eval half: `to_hellinger` (`p ‚Ü¶ ‚ąöp`), `bhattacharyya_distance`, `fit_behavior_manifold` (RBF through node output-distribution centroids in Hellinger space), `trajectory_naturalness` (per-step Bhattacharyya distance of a generation's behavioral trajectory to the behavior manifold), `compute_node_behavior_centroid` / `compute_trajectory_distributions` (model forward passes for the centroids / trajectory).

Persona-manifold + label-form positions: `Manifold.node_roles: list[str | None]` mirrors `node_labels` for role-augmented manifolds (each node's centroid pooled under that role's chat-template substitution). `compute_node_centroid(..., role=, model_type=)` rides the role through `_encode_and_capture_all`. `Manifold.resolve_position(position)` accepts a coord payload (passthrough) or a node-label string (lookup ‚Üí coord tuple; raises `UnknownManifoldLabelError`). `Manifold.nearest_node_index` / `.nearest_node_label` / `.nearest_node_role` short-circuit on label form, otherwise use the domain's chordal distance for the nearest-node geometry. `SteeringManager.add_manifold` calls `resolve_position` before stamping the per-layer entry, so the hook only ever sees coord tuples. The session aggregates manifold-implied roles in `session.steering()`: soft-warn + highest-`|coeff|`-wins on disagreement (`RoleBaselineMismatchWarning`); an explicit `:role-<X>` vector term wins over any implied role with the same warning.

Discover-coord derivation: `derive_pca_coords(centroids, *, max_dim=8, var_threshold=0.70) ‚Üí (coords, PcaDiagnostics)` runs SVD on the centered centroids, picks the smallest prefix whose cumulative variance crosses the threshold (capped at `max_dim`), and projects onto those `k` left-singular directions. `derive_spectral_coords(centroids, *, max_dim=8, k_nn=None, bandwidth=None) ‚Üí (coords, SpectralDiagnostics)` builds a symmetric (union) k-NN graph via `_knn_adjacency`, weights edges with a heat kernel, eigendecomposes the symmetric normalized Laplacian via `torch.linalg.eigh`, drops the smallest eigenvalue, and picks `k` by the eigenvalue-*ratio* heuristic `argmax(őĽ_{k+1}/őĽ_k)` over `[1, max_dim]` ‚ÄĒ the absolute gap őĽ_{k+1} ‚ąí őĽ_k over-picks on S^1 because eigenvalues scale ~k¬≤. Connectivity is enforced with a union-find scan over the adjacency upper triangle (avoids both scipy and the eigenvalue-counting tolerance choice); a disconnected graph raises `ValueError` naming the component count and points the user at `--k-nn` or `--method pca`. `discover_coords(centroids, method, **kwargs)` is the dispatcher used by `ManifoldExtractionPipeline.fit`. Both diagnostics are JSON-serializable scalars + flat number arrays so they ride into the sidecar through `_diagnostics_to_dict` and surface in the CLI / webui inspectors.

`session.generate_statements(concepts, *, scenarios=None, n_scenarios, statements_per_cell, share_moment=False)` is the unified statement-corpus generator ‚ÄĒ folds the former `generate_pairs` (bipolar contrastive pairs) and `generate_concept_statements` (K-tuple discover-mode corpora) into one method. One LLM call writes shared scenarios (anti-allegory clause included), then per-cell calls fill the corpora. Scenario sharing across the row is load-bearing ‚ÄĒ without it the per-concept centroids would mix concept and scenario signal and discover-mode layouts would surface scenario as the dominant axis. `share_moment=False` (default) runs one LLM call per (scenario, concept) cell producing K independent statements; `share_moment=True` runs one LLM call per scenario producing K *moment-shared* groups, each carrying one statement per concept aligned on a concrete moment ‚ÄĒ the bipolar contrastive shape that gives DiM clean within-pair cancellation, generalized to N‚Č§26 concepts. Short cells / short groups are padded with the last entry so the scenario-shared-index invariant survives a flaky LM. Returns `{concept: [statement, ...]}`; the extraction pipeline calls it with `[pos, neg]` + `share_moment=True` and recovers pairs via `zip`, while `vector manifold generate` and `POST /saklas/v1/manifolds/generate` wrap the `share_moment=False` form into a freshly-written discover folder.

## hooks.py

`SteeringHook` carries the per-layer composed state and routes each forward through one of two injection paths. `injection_mode` (`"angular"` default, or `"additive"`) is stamped at construction and can be re-stamped on `recompose`, so flipping the session mode between calls re-warms the cache without rebuilding hooks.

`recompose(additive_entries, ablation_entries, device, dtype, ctx, *, injection_mode=None, theta_max=None)` groups `(tensor, effective_alpha, trigger)` triples by `Trigger` value. Per additive group it builds `composed = ő£ őĪ_i √ó baked_i` and `theta_strength = ||ő£ őĪ_i √ó (baked_i/||baked_i||)||`. The fast path applies when a single `Trigger.BOTH` group has no ablation; under angular, `_refresh_angular_cache` populates Python-scalar `_d_hat`/`_theta`/`_cos_t`/`_sin_t` so the hot path runs zero `.item()` calls.

- **Angular hot path**: per-position Givens rotation (`_angular_inplace`). Decompose `h = h_‚ą• + h_‚ä•`, rotate the unit vector toward `dŐā`, restore the original per-position norm. Norm-preserving by construction ‚ÄĒ no rescale. Near-aligned positions (`||d_perp|| < 1e-6`) get a `torch.where` no-op fallback to dodge shrinkage.
- **Additive hot path**: `vector_norm(fp32) ‚Üí add_(composed) ‚Üí vector_norm(fp32).clamp_(1e-6) ‚Üí mul_(ratio)` ‚ÄĒ two norms, one in-place add, one rescale.

Slow path: ablation groups fire first (`h' = h ‚ąí őĪ(h¬∑dŐā ‚ąí őľ¬∑dŐā)dŐā` per active direction), then active additive groups are summed and rotated/added once. Multi-direction ablation at one layer is naive-parallel ‚ÄĒ exact for orthogonal targets, over-ablates correlated ones (compose `!a + !b|a` for a clean subspace). A pre-check short-circuits when no group fires (e.g. `AFTER_THINKING` during prefill).

`SteeringManager` groups vectors by layer and dispatches per mode at `apply_to_model(...)`. Additive: `effective_alpha = user_alpha √ó _STEER_GAIN`; share is automatic via `||baked_L||`. Angular: pre-computes `share_L = ||baked_L|| / ő£ ||baked||` and passes `effective_alpha = user_alpha √ó share_L` per layer, so cumulative `ő£_L őł_L = |őĪ| √ó őł_max` regardless of layer count. Without this share-weighting the per-layer rotations stack to `N √ó |őĪ| √ó őł_max` and crash coherence ‚ÄĒ keep it. `_STEER_GAIN = 2.0` multiplies **only** under additive mode. `DEFAULT_THETA_MAX = ŌÄ/2`. `add_vector` and `add_ablation` parallel each other; ablation entries whose profile layers are missing from `layer_means` are silently skipped. `add_manifold` registers a manifold-steering term, validating the position-tuple arity against the manifold's domain (the grammar cannot ‚ÄĒ it does not know the domain); `apply_to_model` precomputes the per-layer manifold target at the term's fixed position once (`Manifold.manifold_point`) and attaches a `(basis, mean, target, effective_alpha, trigger)` entry. Manifold groups force the slow path (`all_fast_path` returns False) and run *last* in `hook_fn` via `_apply_manifold_groups`. The operator dispatches on `injection_mode`: angular routes to `subspace_rotate` (Givens rotation of `h_par` toward target by `őĪ ¬∑ őł_max` in the subspace plane, `||h - mean||` preserved, no rescale), additive routes to `subspace_replace` (linear blend of `h_par` toward target, then global norm restore ‚ÄĒ the destructive in-subspace overwrite). Either dominates ablation/additive at the layers it covers. One manifold per layer; overlapping manifolds raise `SteeringExprError`.

`effective_alpha = user_alpha ¬∑ share_L ¬∑ _MANIFOLD_GAIN ¬∑ quality_factor`. The user őĪ is clamped to `[0, 1]`. **Share-weighting**: `share_L = ||eval_rbf(node_params, ..., node_params)||_F / ő£_L'`, the manifold analogue of vector steering's `||baked_L||` weighting ‚ÄĒ recovered from the RBF interpolant's exact-at-nodes property without a separate stored field. Cumulative `ő£_L share_L = 1` makes the budget layer-count-invariant. **Per-mode gain**: `_MANIFOLD_GAIN_ANGULAR = 8.0`, `_MANIFOLD_GAIN_ADDITIVE = _MANIFOLD_GAIN_ANGULAR ¬∑ _STEER_GAIN = 16.0` ‚ÄĒ pins the user-facing őĪ scale into vector-comparable territory (`őĪ ‚Čą 0.5` lands in the coherent persona-expression band on the bundled `personas`). The additive gain is twice the angular per `_STEER_GAIN`, mirroring vector steering's pattern of additive operators needing 2√ó the calibration of angular ones. **Fit-quality normalization** (additive only, v4+ manifolds only): `quality_factor = 1/‚ąömean_EV` where `mean_EV` is the average per-layer explained-variance ratio from `Manifold.explained_variance`; a poorly-fitted manifold (low EV) gets a larger effective őĪ to compensate for the weaker per-őĪ behavioral magnitude. Clamped at `mean_EV ‚Č• 0.01` so a degenerate manifold can't multiply őĪ by ‚ąě. Angular mode skips the EV normalization ‚ÄĒ `subspace_rotate` is direction-only, EV affects only the coherence of the rotation direction, no őĪ scaling helps with that.

`HiddenCapture` ‚ÄĒ session/TUI companion: `attach`/`detach`/`stacked`/`latest_per_layer`. Each capture is `detach().clone()` of `output[0, -1, :]` (device-local). The session sets capture width: `_begin_capture(widen=False)` attaches to the probe-layer union; `widen=True` (driven by `SamplingConfig.return_hidden`) attaches to every layer and pays one device‚Üíhost transfer per layer at finalize.

Incremental mode (`set_incremental(step_sink)`, called after `attach`): the per-layer hook OVERWRITES its bucket (length-1) instead of appending, so device memory stays O(layers¬∑D) for the whole gen instead of growing the full O(T¬∑layers¬∑D) stack. After the highest hooked layer (`_max_layer`) stores its slice ‚ÄĒ last in the forward, since forward hooks fire in layer-execution order ‚ÄĒ the sink fires once with `latest_per_layer()`. `latest_per_layer()` / the streaming tap's `bucket[-1]` reads still return the latest (length-1 bucket). `attach`/`clear` reset incremental state (append default); `set_incremental` must be re-called per gen to opt in. The session opts in via `_begin_capture` when `(not widen) and monitor.probe_names and not manifold_monitor.probe_names` ‚ÄĒ the common monitored case ‚ÄĒ wiring `step_sink = score_single_token_tensor(latest) ‚Üí self._incremental_rows`. `_finalize_generation` then builds `(agg, per_token)` from those rows (`_score_incremental`) instead of `score_captured`, bit-identical to `score_per_token` (same `[:n]` EOS-overshoot trim, same special-token agg walkback, same accumulate/pending semantics). Manifold probes (need the pooled-mean stack) and `return_hidden` (lands the full stack on the result) force full retention.

## monitor.py

`TraitMonitor` scores probes against per-layer hidden states. `_normalize_hidden` mean-centers with layer means and L2-normalizes; per-layer probe weight is `||baked||` (= `share √ó ref_norm`), recovered from tensor magnitude. `_ensure_cache` builds a per-layer stacked cache `{layer: (V[P,D], W[P])}` ‚ÄĒ one matmul per layer scores every probe in one launch, one `.cpu().tolist()` at the end, zero `.item()` calls in the hot path.

Entry points: `score_per_token` (primary; returns `(aggregate, per_token)`, updates history when `accumulate=True`, trims the EOS capture off-by-one so `capture[i] ‚ÜĒ generated_ids[i]`); `measure_from_hidden` (pre-aggregated test path); `measure` (runs a forward pass for arbitrary text); `score_single_token` / `score_single_token_per_layer` (inline per-token, used by the SSE stream and the probe-gate callback); `score_stack` (per-token over a pre-captured `[T,D]` stack, no `generated_ids`/`tokenizer` dependency ‚ÄĒ backs `session.score_hidden`, `accumulate` defaults `False`).

Live running mean for streaming: `begin_live`/`update_live`/`end_live`; `get_current_and_previous` prefers live values so the TUI trait panel ticks during streaming. `add_probe`/`remove_probe` invalidate the stacked cache wholesale.

`ManifoldMonitor` is the read-side peer to `TraitMonitor` ‚ÄĒ manifold probes, not vector probes. `add_probe(name, manifold, *, top_n=DEFAULT_NEAREST_TOP_N)` pre-caches the per-layer node-value tensors in world coords (`(K, D)`) and reduced subspace coords (`(K, R)`) so per-token distance computations are one batched cdist in R-dim per layer; `remove_probe` / `attached_probes` / `attached_layers` mirror the obvious accessors. `score_single_token(hidden_per_layer)` returns `{name: ManifoldTokenReading(fraction, nearest)}` over the shared layer set, EV-weighted across layers ‚ÄĒ fraction is `||h_par_c|| / ||h - mean||` (the share of the centered activation living in the manifold subspace, hot-path: one `decompose` call + two `vector_norm`s per layer per probe), nearest is top-N node labels by EV-weighted distance vote in reduced coords. `flat_scalars(readings)` flattens to `{"<name>:fraction": float, "<name>@<label>": -distance}` ‚ÄĒ the latter signed so "larger = closer" matches the gate-callback convention; the session merges this into `TriggerContext.probe_scores` so `@when:<manifold>:fraction` and `@when:<manifold>@<label>` gates parse-side stuff their namespaced probe name straight into `ProbeGate.probe` and the runtime gate machinery is unchanged. `score_aggregate(captured_per_layer, *, agg_index=None)` is the end-of-gen slow path: pools the **last non-special token** per layer (`agg_index`, the session-supplied `vectors.last_content_index` walkback; default the final row) ‚ÄĒ the same single-state discipline extraction and the vector aggregate use, *not* a mean across the trajectory (that's `score_single_token`). It runs the three channels over the pooled activation, and additionally calls `invert_parameterization` per layer to recover authoring coords + normalized residual. Returns `{name: ManifoldAggregate}` with `fraction_mean / fraction_per_layer / nearest / coords / coords_per_layer / residual_mean / residual_per_layer`. `has_pending_per_token` / `consume_pending_per_token` mirror `TraitMonitor`'s streaming-consumption pattern. Same hot-path discipline as the vector path: no `.item()` per token, fp32 norms throughout, `torch.linalg.vector_norm` + `torch.cdist`. `layer_means` is accepted at construction for parity but unused ‚ÄĒ manifold fraction centers on each fit's per-layer `LayerSubspace.mean`, not the global layer mean.

## triggers.py

`Trigger` (frozen dataclass) carries phase flags plus an optional `gate: ProbeGate | None`. `Trigger.active(ctx)` consults the phase flags and, when a gate is set, `ctx.probe_scores[gate.probe]` against `score <op> threshold`. `Trigger.first(n)` / `Trigger.after(n)` / `Trigger.when(probe, op, threshold)` are the factories; `when` builds the canonical `prompt=False, gate=ProbeGate(...)` shape. `TriggerContext.probe_scores` is filled by the per-step score callback during decoding and cleared on `reset()`. Gated triggers report inactive during prefill (no reading yet) and for missing probes (no raise). `ProbeGate` is frozen/hashable so identical gates compose under equality in `recompose`'s per-trigger grouping. `ProbeGate.probe` is the canonical scalar key from whichever monitor supplied it ‚ÄĒ `"angry.calm"` from `TraitMonitor`, `"circumplex:fraction"` / `"circumplex@elated"` from `ManifoldMonitor.flat_scalars`. The runtime gate code is the same lookup either way; only the parser knows the difference.

## hooks ‚ÜĒ generation: cuda_graphs.py

CUDA-graphs / `StaticCache` support. When `cuda_graphs=True` + device is CUDA + a StaticCache-compatible architecture + fast-path-eligible steering, generation routes through `transformers.StaticCache` so kernel shapes stay fixed and `torch.compile` can capture graphs. `is_cuda_graphs_supported(model, device)` probes viability and caches `(supported, reason)` keyed by underlying module id (through `torch.compile`'s `_orig_mod`), device, and dtype ‚ÄĒ so a pre-compile probe and a post-compile call hit the same entry. `make_static_cache` builds a cache sized to `prompt_len + max_new_tokens + offset`. `warn_once` logs the fallback reason once per model. Slow-path steering (probe gates, multi-trigger, ablation under CTX mutation) bypasses StaticCache ‚ÄĒ the eligibility check lives at the steering layer.

## generation.py

Token-by-token decode + KV cache, wrapped in `torch.inference_mode()`. Models that return no `past_key_values` during prefill (custom modeling that ignores the cache kwarg, e.g. talkie) flip `no_cache_mode` and re-feed the full accumulated input each step ‚ÄĒ O(N¬≤), one-time warning, off the throughput-invariant path; a preallocated buffer + advancing view avoids per-step `torch.cat`.

`GenerationConfig` (frozen) holds session-level defaults (`max_new_tokens`, `temperature`, `top_p`, `top_k`, `system_prompt`); per-call `SamplingConfig` overrides are composed into a local copy and never mutate `session.config`. Per-message role-substitution labels ride `SamplingConfig.user_role`/`assistant_role` (like `seed`) ‚ÄĒ the roleplay scaffold. Top-p is via `torch.topk`, not full-vocab sort; `top_k` (default `None` ‚Üí 1024 cap) is a hard candidate-pool cap applied **before** top-p (llama.cpp/Ollama order). In-place ops throughout; MPS sync at end-of-loop prevents Metal command-buffer reuse crashes.

`generate_steered(...)` accepts `seed`, `stop`, `logit_bias`, `presence_penalty`/`frequency_penalty` (sparse device-side unique-id/count buffers, applied to raw logits before temperature), `logprobs` (0 = chosen-only, >0 = top-k), `score_callback`, `forced_prefix`, and `steering_active` (default `True`; the session passes `bool(self._steering.hooks)` so the per-token `nan_to_num_`/`clamp_` fp16-overflow guard ‚ÄĒ two vocab-width kernels ‚ÄĒ runs only when a steering hook is attached, skipping it on the unsteered path). `score_callback` enables probe-gated triggers: when set, it's called once per forward (after the model returns, before the next hook fire) and its dict is written to `trigger_ctx.probe_scores`. The session wires it only when the active steering carries a gated `Trigger`; otherwise the kwarg stays `None` (one `is None` check per step).

`forced_prefix` is a list of token ids forced for the first `len(forced_prefix)` decode steps instead of the sampled token. The `multinomial` draw still runs every step, so re-seeding keeps the RNG stream bit-identical through the fork point. `session.fork_from_token(node_id, raw_index, alt_token_id)` slices a source assistant node's `raw_token_ids` and lands a sibling under the same recipe; `session.prefill_assistant(node_id, text, *, steering, sampling, on_token)` tokenizes `text` (`add_special_tokens=False`) into the prefix and lands a sibling assistant under a *user* node, always `thinking=False`. Both bump the token budget by the prefix length. Legacy/transcript-loaded nodes carry no `raw_token_ids` and aren't forkable.

`session.append_user_turn(parent_node_id, text, *, role_label=None)` / `session.append_assistant_turn(user_node_id, text, *, role_label=None)` are the no-generation "commit" entry points (Ctrl+Enter on either surface). Neither runs a decode ‚ÄĒ they land a turn directly on the tree and advance the active node. `role_label` stamps the per-turn role-substitution label, same as a generated turn. `append_user_turn` wraps `tree.add_user_turn` (same dedup, refuses anchoring under a user-role parent via `_check_user_send_target`); `append_assistant_turn` tokenizes `text` into `raw_token_ids` so the authored turn stays forkable, lands the node with `recipe=None` (the implicit "no model run produced this" marker, same shape transcript-loaded nodes carry), and clears `tokens`/`thinking_tokens` since authored turns have no per-token scores. Both raise `InvalidNodeOperationError` on empty text or shape violations.

`detect_base_model(tokenizer)` is the named form of the `tokenizer.chat_template is None` check (canonical across `vectors.py` / `build_chat_input`) ‚ÄĒ a base/completion model has no chat template, so frontends branch on it to switch to a raw-completion UI. `session.is_base_model` exposes it as a property.

Per-message roles (roleplay scaffold): each turn carries its own substitution label, stamped at send time and rendered faithfully thereafter. `LoomNode.role_label` holds the label (`None` = standard role); `tree.messages_for(with_labels=True)` surfaces it under a `"label"` key. `build_chat_input(..., gen_role=, model_type=)` takes per-message `label` keys plus `gen_role` (the generation-prompt assistant label) and routes to `role_templates.apply_with_per_turn_roles` when any label is present (zero-overhead `apply_chat_template` passthrough otherwise ‚ÄĒ so `cloning.py`/`joint_logprobs.py`/scenario-gen callers are unaffected). `apply_with_per_turn_roles` renders once with standard labels then **positional-splices** each header's occurrences in render order against the per-turn label list (assistant occurrences = history assistant turns + the trailing `gen_role`); best-effort/lenient so gpt-oss's bare `<|start|>assistant` generation prompt gracefully no-ops `gen_role` (the pre-existing gpt-oss assistant gap ‚ÄĒ user-side and history-assistant relabels still land). `role_templates.py` carries the two per-family header registries (`ROLE_HEADERS` / `USER_ROLE_HEADERS`); only gpt-oss's user header diverges (`<|message|>` vs the assistant `<|channel|>`, verified against the real template). `_chat_input_cache_key` folds in `gen_role` + the per-message label tuple so role-tagged renders never collide. `apply_with_role` (global single-side splice) stays for role-augmented *extraction* (`vectors.py`, `role=`).

`session._prepare_input(..., user_role=, assistant_role=)` builds the messages with per-node labels from the tree (history) + this send's `user_role` on the new user turn, and resolves the generation-prompt label as `_active_role` (the steering-driven role from `:role-<slug>` vectors / persona manifolds, set transiently by `_SteeringContext`) when set, else this send's `assistant_role`. Steering wins for the *render* (so steer baseline matches extract baseline) but the *stamped* node label is the box value (steering is transient, not persisted). `_start_loom_assistant` stamps `sampling.user_role`/`assistant_role` onto the created user / assistant nodes; raw / flat mode carries no labels.

Flat (base-model / completion) generation rides the `raw=True` kwarg through `generate`/`generate_stream`. `_prepare_input` with `raw` + a string skips the chat template entirely: it feeds the model `tree.flat_text(parent) + input` ‚ÄĒ the active-path node texts concatenated verbatim, no roles, no markers (`stateless` drops the tree walk so the buffer is just `input`). `_start_loom_assistant` with `raw` doesn't pair turns: a non-empty `input` is recorded as its own node (no `_check_user_send_target` ‚ÄĒ in flat mode the role tag is authorship provenance, not a turn-taking constraint), an empty `input` is a bare continuation that begins an assistant straight off the active leaf. So a raw continuation chains `assistant`-under-`assistant` freely, and the GUI's non-linear edit model (every buffer edit branches a fresh span) maps onto the tree without a new node type.

`supports_thinking` delegates to `_detect_think_delimiters`, which round-trips an assistant message through the chat template and returns `(start_id, end_id, response_start_id, starts_in_thinking)` ‚ÄĒ works across Qwen/Gemma via the `enable_thinking` template round-trip, and falls back to two literal-string detectors when no `enable_thinking` switch exists: `_detect_channel_delimiters` for gpt-oss-style `<|channel|>‚Ä¶<|message|>`, then `_detect_bracket_delimiters` for Mistral-3-Reasoning-style `[THINK]‚Ä¶[/THINK]` (where the reasoning instruction is baked into the template's default system prompt, so the state machine must engage regardless of the `thinking` flag to keep the section out of the response). `ThinkingState` enum (`IDLE ‚Üí PREAMBLE ‚Üí THINKING ‚Üí RESPONSE_PREAMBLE ‚Üí RESPONSE ‚Üí DONE`) is set at every transition and forced to `DONE` in `finally`. `GenerationState` holds `stop_requested` (Event), `token_queue` (SimpleQueue), `thinking_end_idx`, `thinking_state`, `finish_reason` (`"stop"` default, flips to `"length"`/`"stop_sequence"`), `emit_map` (maps emitted tokens back to `generated_ids` space), and `response_text` (authoritative final non-thinking text, since stop sequences can trim a partial token).

`ÔŅĹ` partial-UTF-8 tokens are marked `None` in the token table and buffered until complete. `_get_token_table` uses chunked (8192) `batch_decode` with per-chunk fallback to per-id `decode()`. `_tok_key` is the shared cache-key helper for `_eos_cache` / `_token_table_cache` / `_think_delim_cache`. The `None` done sentinel is **not** emitted by `generate_steered` ‚ÄĒ the session worker / TUI closure puts it on the queue in `finally`.

## session.py

`SaklasSession` owns the model, profile registry (`_profiles`), monitor, `SteeringManager`, `HiddenCapture`, generation defaults (`session.config`), the loom tree, and a synchronous `events: EventBus`.

Construction: `from_pretrained(model_id, *, device, dtype, quantize, probes, injection_mode="angular", theta_max=None, ...)` does HF load + probe bootstrap + layer-mean compute; `__init__(model, tokenizer, ...)` accepts a pre-loaded model. `theta_max=None` ‚Üí `DEFAULT_THETA_MAX`. Both paths unconditionally call `materialize_bundled()` + `selectors.invalidate()` early ‚ÄĒ `bootstrap_probes` does this transitively but is skipped when `probes=[]`, which would otherwise leave fresh bundled concepts invisible to `_all_concepts()`.

`generate` / `generate_stream` are keyword-only and accept `steering: str | Steering | None` (expression string or pre-built `Steering`; dicts rejected), `sampling: SamplingConfig`, `thinking=None` (auto-detect via `supports_thinking`), plus loom args (`parent_node_id`, `n`, `recipe_override`). Both return `RunSet`; `.first` is the underlying `GenerationResult` and common attributes delegate to it. `generate_stream(live_scores=False)` suppresses inline per-token scoring but still computes final per-token scores.

`session.steering(value)` is a context manager: coerces via `Steering.from_value`, materializes `ProjectedTerm` entries into derived profiles, and pushes onto a LIFO stack. Per-call `injection_mode`/`theta_max`/`projection_metric` ride a parallel `_steering_override_stack` (triplets); `_resolve_projection_metric` / the resolver walks it top-down for inner-wins. `_resolve_pole_aliases` keeps a cache-hit auto-load fast path (`_try_autoload_vector`) so HTTP clients can steer bundled probes without pre-registration. Default projection metric is Mahalanobis ‚ÄĒ `_materialize_projections` passes `session.whitener` to `project_profile`; whitener-coverage misses fall back to Gram-Schmidt transparently.

No persistent steering hooks. `generate`/`generate_stream` wrap `_generate_core`, which owns the `_gen_lock` re-entry guard, `_gen_phase` lifecycle (`IDLE`/`PREAMBLE`/`RUNNING`/`FINALIZING`, read via `session.gen_state`), the steering context, `_begin_capture`/`_end_capture`, `_finalize_generation`, and deterministic teardown. Helpers split the mechanics: `_prepare_generation_call`, `_start_loom_assistant`, `_snapshot_steering_alphas`, `_run_generation_loop`. `generate_stream` spawns a worker enqueueing `TokenEvent`s into a `SimpleQueue`; iterator `close()` triggers the same teardown.

Monitor scoring is in-flight: `HiddenCapture` is attached on the probe-layer union, `_finalize_generation` ‚Üí `monitor.score_per_token`, no second forward pass. Hidden-state round-trip: `SamplingConfig(return_hidden=True)` widens capture to every layer and lands `GenerationResult.hidden_states` (`dict[int, Tensor]`, CPU); `session.score_hidden(hidden, *, per_token, accumulate)` re-scores any compatible dict.

Manifold probes ride the same plumbing as a peer monitor. `session.add_manifold_probe(selector, *, as_name=None, top_n=3)` resolves the selector through `_ensure_manifold_loaded` (the same lazy-load path manifold steering uses) and registers on `self._manifold_monitor`; `remove_manifold_probe(name)` detaches. `_begin_capture` widens the captured layer set to the union of vector-probe layers and `manifold_monitor.attached_layers()` so a single `HiddenCapture` feeds both monitors. The gate score callback (`_build_gating_score_callback`) calls `TraitMonitor.score_single_token` and (when any manifold probe is attached) `ManifoldMonitor.score_single_token` + `flat_scalars`, merging both into the `dict[str, float]` written to `TriggerContext.probe_scores`. `_finalize_generation` reads the captured stack one more time (post-`_end_capture`, since `_per_layer` survives the `detach()`) and calls `ManifoldMonitor.score_aggregate` to populate `GenerationResult.manifold_readings`. `generate_stream` emits `TokenEvent.manifold_readings` per token when `live_scores=True` and manifold probes are attached, alongside the vector-probe `scores`.

`session.lock` is an `asyncio.Lock` (server-owned FIFO serializer), distinct from the threading `_gen_lock`. `generate_batch` / `generate_sweep` return `RunSet` (list-like, plus `node_ids`/`grid`/`.to_collector()`/`.to_dataframe()`); `generate_sweep` builds the Cartesian product of `{concept: [alphas]}` and lands the runs as loom siblings under one user turn. Events from the hot path: `GenerationStarted`, `SteeringApplied`, `SteeringCleared`, `ProbeScored`, `GenerationFinished`, plus `VectorExtracted` / `ManifoldExtracted`; server/TUI subscribers must hop via `loop.call_soon_threadsafe`. `session.extract_manifold(folder, ...)` delegates to `ManifoldExtractionPipeline` (gated on `GenState.IDLE` like `extract`); manifold artifacts load lazily into `self._manifolds` via `_ensure_manifold_loaded` on steering-scope entry.

The custom-concept extraction pipeline lives in `extraction.py`; the session delegates to `ExtractionPipeline`. `_N_PAIRS = 45`, `_N_PAIRS_PER_SCENARIO = 5`.

## loom.py

`LoomTree` ‚ÄĒ the engine-side conversation tree. Nodes are turns, children are alternative continuations; the active path is the model's context. Owns the data model only ‚ÄĒ generation integration, gen-lock concurrency, persistence, and event delivery live in `session.py`/the server. `messages_for(leaf)` renders the path as v2 `{role, content}` chat messages (`with_labels=True` adds each node's per-turn `role_label` under a `"label"` key for the per-turn role render); `flat_text(leaf)` is the base-model analogue ‚ÄĒ the same path joined as one continuous string, no roles, no separators, synthetic root skipped (the raw-generation prefix). `LoomNode.role_label` is the per-turn role-substitution label (roleplay scaffold, stamped at send via `add_user_turn`/`begin_assistant`'s `role_label=`, persisted in `to_dict`/`from_dict`). Node ids are 26-char ULIDs (inline `_ulid`, sortable by timestamp prefix). Five primitives: edit, branch, navigate, delete_subtree, plus `regenerate` (via `generate(parent_node_id=...)`). `Recipe` is the per-node reproducibility receipt (steering expression, sampling, thinking, seed, probe set + per-probe sha256); `Recipe.overlay`/`invert_steering`/`compose_modifier` back the auto-regen modes (`unsteered`/`inverted`/`reseed`/`cool`/`hot`/custom). Per-node token blobs are in memory during streaming; `to_dict` omits them, `save` writes them to a gzip sidecar. Mutators raise `MutationDuringGenerationError` (HTTP 409) when conflicting with an in-flight gen; `UnknownNodeError` (404) and `InvalidNodeOperationError` (400) cover the rest.

## tree_filter.py

Filter grammar for tree pruning ‚ÄĒ deliberately distinct from the steering `@when:` grammar (that gates per-step readings; this gates per-node aggregates). Clauses are `<agg>:<probe> <op> <threshold>`; `agg_op` is `agg` (default, `LoomNode.aggregate_readings`), `any` (max/min over per-token scores), or `last`; multi-clause is AND. `parse_filter` ‚Üí `FilterClause`; `FilterClause.evaluate(node, *, per_token_scores=)`; `filter_tree(tree, text, ...)` is the free-function form behind `LoomTree.filter_by_expr`. Missing probe ‚Üí clause is `False`. `FilterParseError` on any parse problem.

## loom_diff.py

Cross-branch diff primitives. `text_diff` ‚ÄĒ word-level diff over whitespace-split tokens via `difflib.SequenceMatcher`, returns aligned `DiffSpan`s. `readings_diff` ‚ÄĒ per-probe `őĒ = b ‚ąí a` table sorted by `abs(delta)`, missing-side readings default `0.0` with `a_value`/`b_value` preserved. `per_token_diff` ‚ÄĒ byte-offset alignment between two token sequences, stops where bytes diverge. `steering_delta` ‚ÄĒ compact `"+0.2 calm"` edge label by walking both expressions through the shared grammar.

## transcript.py

Transcript export/import for loom paths. A `Transcript` is a saved path through the tree (system prompt + user turns + each assistant turn's `Recipe` + final readings), serialized to YAML via `to_yaml`/`from_yaml`. `SAKLAS_TRANSCRIPT_VERSION = 1`. Three import modes: **default** (new top-level branch off root), **here** (child of active node), **merge** (attach the non-matching tail at the deepest matching user-turn prefix). Guards: model mismatch refuses `--merge` (`TranscriptModelMismatch`); system-prompt mismatch warns + banners; missing probes warn; probe hash drift warns, or raises `TranscriptProbeDriftError` under `--strict`.

## joint_logprobs.py

Cross-branch joint-logprob computation for the loom NodeCompareDrawer. Given two sibling assistant nodes, force-replays each branch under its stamped recipe and assembles per-aligned-position rows carrying both branches' chosen-token logprobs *and* the cross-branch evaluation. `_compute_rows` is the pure-tensor inner loop ‚ÄĒ alignment via `per_token_diff` + sampler-logprob lookup at `prefix_len + i ‚ąí 1`. `compute_joint_logprobs(session, a_id, b_id)` is the IO wrapper ‚ÄĒ reconstructs the prefix (shared prefix via `_shared_prefix_len`, not re-tokenization), force-replays through the model, then calls the inner loop. `approx_kl` is top-K-truncated `KL(P_A‚ÄĖP_B)` ‚ÄĒ approximate signal, the tail is unobserved. Results cache lazily on `session._joint_logprob_cache` keyed by sorted `(a_id, b_id)`; the route layer flips a/b labelling via `reorient_for_request`. Held under the session lock ‚ÄĒ the forward passes compete with generation.

## results.py

`GenerationResult`, `RunSet`, `TokenAlt`, `TokenEvent`, `ProbeReadings`, `ManifoldTokenReading`, `ManifoldAggregate`, `ResultCollector`. `RunSet` is the standard multi-run shape: list-like, plus `node_ids`, `grid`, `.first`, `.to_collector()`, `.to_dataframe()`. `TokenAlt` is a frozen `(id, text, logprob)` triple ‚ÄĒ engine decodes text at capture time so consumers don't re-tokenize. `TokenEvent` carries `thinking`, `logprob`, `top_alts`, `finish_reason`, `scores` (per-probe cosine sims, populated by `generate_stream` only when probes are active and `live_scores=True`), `perplexity`, `manifold_readings` (per-attached-probe `ManifoldTokenReading`, same live-stream gate). `GenerationResult` carries `prompt_tokens`, `finish_reason`, optional `logprobs`, `manifold_readings` (per-attached-probe `ManifoldAggregate` populated in `_finalize_generation`). `applied_steering` holds the canonical steering expression that produced the result (round-trips through `parse_expr`). `ManifoldTokenReading` is the per-token shape (`fraction`, `nearest`); `ManifoldAggregate` is the end-of-gen shape (`fraction_mean / fraction_per_layer / nearest / coords / coords_per_layer / residual_mean / residual_per_layer`). Both have `to_dict()`. `ResultCollector` handles batch export (JSONL/CSV/DataFrame). `to_dict()` omits `hidden_states` (tensors don't serialize ‚ÄĒ persist with `torch.save`).

## histogram.py

Shared per-layer magnitude-histogram helper. `HIST_BUCKETS = 16`; `bucketize(norms, buckets)` collapses sorted per-layer norms into evenly-sized groups (one bucket per layer when there are fewer layers than buckets). Used by the TUI WHY footer and CLI `vector why`.

## sampling.py / steering.py / steering_expr.py / events.py / errors.py / profile.py

`SamplingConfig` ‚ÄĒ per-call frozen config with `merged_with`. `Steering` ‚ÄĒ frozen; `from_value` accepts `str | Steering | None` (strings route through the shared parser, dicts rejected); optional `injection_mode`, `theta_max`, `projection_metric` (all `None` = inherit session default). `events.py` ‚ÄĒ synchronous `EventBus` + 6 event dataclasses. `errors.py` ‚ÄĒ `SaklasError` base; every saklas exception multi-inherits through it while keeping its stdlib MRO, so `except SaklasError` catches the family and `except ValueError`/`RuntimeError` at existing sites still works. `user_message()` returns an `(http_status, text)` pair. `profile.py` ‚ÄĒ `Profile` wraps `dict[int, Tensor]` with `.layers`/`.weight_at`/`.save`/`.load`/`.to_gguf`/`.merged`/`.merged_with`/`.promoted_to`/`.cosine_similarity`/`.projected_away`; `cosine_similarity(other, *, per_layer=False, whitener=None)` is magnitude-weighted; empty layer intersection raises `ProfileError`.

`steering_expr.py` hosts the unified grammar every input surface shares. `parse_expr(text, *, namespace=None)` ‚Üí `Steering`; `format_expr(steering)` renders it back (round-trips); `referenced_selectors(text)` yields `(ns, concept, variant)` triples for install-time checks. Projection terms land in `Steering.alphas` as `ProjectedTerm(coeff, trigger, operator, base, onto)`, materialized into derived profiles by the session before the hook layer sees them. `!atom` produces an `AblationTerm` (default coeff 1.0 ‚ÄĒ fully replace with the neutral mean); it doesn't compose with `~`/`|`. Ablation terms are dispatched separately ‚ÄĒ `normalized_entries()` skips them and they route to `SteeringManager.add_ablation`.

Probe gates (`@when:<probe><op><threshold>`) accept three probe-identifier shapes. Vector probes: `IDENT` optionally followed by `.IDENT` for a bipolar concept (e.g. `@when:angry.calm > 0.4`). Manifold subspace-fraction: `<manifold>:fraction` (e.g. `@when:circumplex:fraction > 0.5`) ‚ÄĒ the `:fraction` suffix is the only fraction-channel slug today, anything else raises `SteeringExprError` at parse time. Manifold label-similarity: `<manifold>@<label>` (e.g. `@when:circumplex@elated > -0.1`) ‚ÄĒ labels with negative thresholds are the common case, since `flat_scalars` stores `-distance` so larger means closer. All three shapes store the namespaced string verbatim in `ProbeGate.probe`, so the runtime gate machinery is identical to vector probes; the parser is the only place the discrimination lives. `format_expr` round-trips by emitting `ProbeGate.probe` unchanged.
