Overall Layout Strategy

How should the TUI divide screen space between chat, controls, and monitoring?

A

Three-Column Layout

Left: Narrow vector/controls panel (~25%) • Center: Chat (~50%) • Right: Trait monitor (~25%)

Maximizes information density. All three concerns visible simultaneously. Chat gets the lion's share of width. Vectors and monitoring flank it.

B

Two-Column + Tabbed Right Panel

Left: Chat (~60%) • Right: Tabbed panel (~40%) with tabs for [Vectors] [Monitor] [Config] [Stats]

Wider chat area. Right panel switches context via tabs. Only one concern visible at a time but with more room for each. Good for smaller terminals.

C

Three-Column with Collapsible Panels

Same as A but each column can be toggled on/off via keybindings (F1/F2/F3), allowing the user to dynamically allocate space. Collapsed panels show a thin 1-char sidebar with a label.

Maximum flexibility — full-width chat when you want it, full monitoring when tuning. Best for power users. Can start with all three open and collapse as needed.
